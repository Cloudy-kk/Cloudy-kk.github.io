let 声明

let 跟 var 的作用差不多，但有着非常重要的区别。最明显的区别是，let 声明的范围是块作用域，而 var 声明的范围是函数作用域。块作用域是函数作用域的子集，因此适用于 var 的作用域限制同样也适用于 let。

let 声明的变量不会在作用域中被提升

在解析代码时，JavaScript 引擎也会注意出现在块后面的 let 声明，只不过在此之前不能以任何方式来引用未声明的变量。在 let 声明之前的执行瞬间被称为“暂时性死区”（temporal dead zone），在此阶段引用任何后面才声明的变量都会抛出ReferenceError。

全局声明

与 var 关键字不同，使用 let 在全局作用域中声明的变量不会成为 window 对象的属性（var 声明的变量则会）。

NaN

NaN—-> Not a Number
NaN 的操作始终返回 NaN（如 NaN/10）
isNaN()函数。该函数接收一个参数，可以是任意数据类型，然后判断
这个参数是否“不是数值”。把一个值传给 isNaN()后，该函数会尝试把它转换为数值。某些非数值的值可以直接转换成数值，如字符串”10”或布尔值
console.log(isNaN("blue")); // true，不可以转换为数值

原型

创建的函数，浏览器解析器会向函数中添加一个属性prototype(唯一的)
这个属性对应这一个对象，

当函数作为普通函数调用的时候prototype没有任何作用
当函数以构造函数的形式调用的时候，它所创建的对象都有一个隐含的属性指向该构造函数的原型对象
可以通过__proto__来访问属性


原型对象就相当于一个公共的区域，所有同一个类的实例都可以访问到这个原型对象，我们可以将对象中公有的内容，统一设置到原型对象中
当我们访问一个属性或者方法的时候，它会先在对象自身中寻找，如果有则直接使用，如果没有就回去原型对象中寻找。

  1. 类的所有实例都可以访问
  2. 不会污染全局

所以上面代码改进之后

所以创建构造函数的时候，可以将这些对象共有的属性和方法，统一添加到构造函数的原型对象中。（类似于静态属性，静态方法）

区别直接写在类中
直接写在构造函数中，每次创建对象的时候都要创建改方法，浪费内存
写在原型对象中，所有的(构造函数)类的实例都可以访问，不占空间并且安全

原型对象也是对象。所以它也有原型对象
直到找到Object对象的原型。Object对象的原型没有原型，如果在Object中依然没有找到，则返回undefined（原型链）
